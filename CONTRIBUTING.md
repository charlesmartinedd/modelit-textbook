## üìú Code of Conduct

This project follows a code of conduct that all contributors are expected to follow:
- Be respectful and inclusive
- Focus on educational value and accuracy
- Maintain age-appropriate content (Grades 5-8)
- Ensure NGSS alignment

## ü§ù How to Contribute

### Types of Contributions

We welcome various types of contributions:

1. **Content Creation**
   - Writing new chapters
   - Improving existing chapters
   - Creating activities and exercises
   - Developing assessment questions

2. **Visual Assets**
   - Educational illustrations
   - Diagrams and charts
   - Infographics

3. **Technical Improvements**
   - Build scripts and automation
   - PDF rendering improvements
   - Accessibility enhancements

4. **Documentation**
   - Improving README
   - Writing guides
   - Creating examples

### Getting Started

1. **Fork the repository**
2. **Create a branch** for your contribution (`git checkout -b feature/your-contribution`)
3. **Make your changes**
4. **Test your changes** (if applicable)
5. **Submit a pull request**

## üìù Content Guidelines

### Educational Standards

- **NGSS Alignment:** All content must align with Next Generation Science Standards for middle school
- **Grade Level:** Content should be appropriate for Grades 5-8
- **Accessibility:** Ensure content is accessible to diverse learners

### Content Requirements

- **Accuracy:** All scientific content must be factually accurate
- **Clarity:** Use age-appropriate language and clear explanations
- **Engagement:** Include real-world examples and hands-on activities
- **Inclusivity:** Represent diverse perspectives and experiences

### Chapter Structure

Each chapter should include:

1. **Introduction** - Hook and learning objectives
2. **Core Content** - Main concepts and explanations
3. **Activities** - Hands-on learning experiences
4. **Real-World Connections** - Practical applications
5. **Review Questions** - Assessment and reflection
6. **Key Terms** - Vocabulary definitions

## üé® Style Guide

### Typography

- **Body Text:** 14pt Georgia/Palatino serif
- **H1 Headers:** 42pt
- **H2 Headers:** 28pt
- **H3 Headers:** 20pt

### Color Scheme

- **Primary Blue:** #0F6ACE
- **Cyan Accent:** #48d2fc
- **Text:** Black (#000000) or dark gray (#333333)
- **Background:** White (#FFFFFF)

### Image Guidelines

- **Format:** High-resolution images suitable for print
- **Style:** Educational, age-appropriate illustrations
- **Embedding:** Base64-encoded for PDF reliability
- **Alt Text:** Descriptive alt text for accessibility

### Writing Style

- **Tone:** Engaging, clear, and age-appropriate
- **Voice:** Active voice preferred
- **Length:** Appropriate for middle school attention spans
- **Examples:** Use concrete, relatable examples

## ‚úÖ Quality Standards

### Content Quality Checklist

- [ ] NGSS standards alignment verified
- [ ] Age-appropriate language and concepts
- [ ] Factual accuracy verified
- [ ] Clear learning objectives
- [ ] Engaging activities included
- [ ] Real-world connections present
- [ ] Accessibility considered

### Technical Quality Checklist

- [ ] PDF renders correctly
- [ ] Images display properly
- [ ] Typography consistent
- [ ] Color scheme applied correctly
- [ ] File size reasonable (< 15 MB per chapter)
- [ ] Print-ready formatting

### Review Process

All contributions go through:

1. **Initial Review** - Content and style check
2. **Technical Review** - Formatting and quality verification
3. **Educational Review** - NGSS alignment and pedagogical soundness
4. **Final Approval** - Publisher review

## üì§ Submission Process

### Before Submitting

1. **Review Guidelines** - Ensure your contribution meets all guidelines
2. **Self-Check** - Run through quality checklists
3. **Test** - Verify PDF renders correctly (if applicable)
4. **Document** - Update relevant documentation

### Pull Request Template

When submitting a pull request, please include:

- **Description:** What you've changed and why
- **Type:** Content, technical, documentation, etc.
- **Chapter:** Which chapter(s) affected
- **Testing:** How you tested your changes
- **Checklist:** Completed quality checklists

### Review Timeline

- **Initial Review:** 2-3 business days
- **Revisions:** As needed
- **Final Approval:** 1-2 business days

## üêõ Reporting Issues

4. **Suggest Fix** - If you have a solution, include it
